# Таблица:

1. Пробелы в данных в дате и балансе (null) вывожу как "-". При сортировке по этим полям вывожу такой баланс как минимальный, а дату как наиболее давнюю.
2. Дубли не скрываю, а сортирую рядом с оригиналом и помечаю как дубль: в реальной CRM важно видеть сколько у тебя дублей, чтобы вовремя инициировать объединение сущностей. Иначе могут возникнуть казусы по типу спама одному и тому же контакту сообщения при рассылке из-за того, что у него много дублей.
3. Добавил фильтр по статусам отдельной кнопкой.
4. Добавил колонку VIP.
5. Реализовал поиск клиентам, решил искать не только по имени, но и по email. Вместо простого includes() реализовал fuzzy поиск (учёт опечаток и неточного запроса) с использованием библиотеки Fuse.js. Например, запрос "aLiCceS" даст Alice Johnson. Точность и гибкость поиска можно настроить в поле threshold при подключении.
6. Общая последовательность обработки данных: сначала фильтрация по статусу, затем по поиску, затем сортировка.

# Архитектура

1. **Stores/** Вынес управление состоянием в отдельные stores:
   - `clientsStore.ts` - хранилище клиентов с derived store для отфильтрованных и отсортированных данных
   - `filterStore.ts` - состояние фильтра по статусам
   - `sortStore.ts` - состояние сортировки
   - `searchStore.ts` - состояние поиска

2. **Hooks/** Создал переиспользуемые hooks для инкапсуляции логики:
   - `useClients.ts` - работа с запросом данных клиентов
   - `useFilter.ts` - управление фильтром
   - `useSort.ts` - управление сортировкой
   - `useDuplicateDetection.ts` - определение дублей
   - `useSearch.ts` - управление поиском

3. **Utils/** - Разбил логику на отдельные helper функции:
   - `sortHelpers.ts` - функции сортировки
   - `filterHelpers.ts` - функции фильтрации
   - `searchHelpers.ts` - функции поиска
   - `constants.ts` - глобальные константы

4. **Types/** - Вынес типы:
   - `types/filter.ts` - типы фильтрации
   - `types/sort.ts` - типы сортировки

5. **Components/**:

   **ClientsTable/** - компоненты таблицы:
   - `ClientsTable.svelte` - основной компонент таблицы
   - `ClientsTableHeader.svelte` - заголовок с сортировкой
   - `ClientsTableRow.svelte` - строка таблицы с данными
   - `StatusBadge.svelte` - визуальный бейдж статуса
   - `DuplicateIndicator.svelte` - индикатор дублей

   **Filter/** - компоненты фильтрации:
   - `StatusFilter.svelte` - объединяющий компонент фильтра
   - `FilterButton.svelte` - кнопка с текстом фильтра
   - `FilterDropdown.svelte` - выпадающее меню опций

   **UI/** - переиспользуемые UI компоненты:
   - `Button.svelte` - универсальная кнопка с вариантами
   - `Table/Table.svelte`, `TableHeader.svelte`, `TableRow.svelte` - компоненты таблицы
   - `SearchInput.svelte` - строка поиска.

   **Layout/** - компоненты раскладки:
   - `Content.svelte` - обертка контента
   - `ContentHeader.svelte` - заголовок с действиями
   - `Header.svelte` - хедер приложения
   - `Footer.svelte` - футер приложения

6. **Routes** - Создал основную страницу приложения:
   - `routes/ClientsPage.svelte` - главная страница со всеми компонентами
